# Chroma 56

Wong et al. (2025) の成人絶対音感訓練を参考にしたWebアプリです。1音版と2音版をトップページから選べます。出題音には `smplr` の `SplendidGrandPiano` を使用します。

## オンラインで使う

https://toshiotawa.github.io/chroma-56/

## 起動

`index.html` をブラウザで直接開くか、このフォルダでローカルサーバーを起動します。

```sh
python3 -m http.server 4173
```

その後、`http://localhost:4173` を開いてください。

## 構成

### 共通

- **Day** = 何を解禁するか（新規音・新規ペア）
- **Section** = 新規 / 直近 / 累積 / OOB の配分、Shepard、フィードバックの有無
- セクション3〜5の前に Shepard toneを挿入
- セクション5はフィードバックなし
- 回答は1オクターブ鍵盤（対象音のみ操作可能）と従来の音名ボタンを切替可能
- 鍵盤は回答入力専用で、鍵盤操作による音の再生はなし
- テスト用にセクションスキップ・途中JSONエクスポートあり

### 1音バージョン

- 1日 **300問**（60問 × 5セクション）、Shepard toneは20秒
- 基礎59日 + 最終8日 = 67日
- Fから始め、E、F#、D#、G、D、G#、C#、A、C、A#、Bの順に追加
- 最初のFは「記憶・境界・速度・定着」の4日で学習
- E以降は「集中・記憶・境界・速度・定着」の5日で学習し、集中Dayでは新規音とOOBだけを選択肢に表示
- 最終8日は全12音の統合・汎化・確認（定着型の5セクション構成）
- 新規音を中心に焼き付け、Day 2で直近比較、Day 3で累積復習、Day 4で全体確認
- OOBは新規音の ±1/±2 未習音を優先（円環対応）
- 3音域を均等に組んだデッキをシャッフルし、固定のピアノ音色で出題
- 確認セクションでは前問とのMIDI差が12半音超になる候補を優先

### 2音バージョン

- 固定・比較は **30問 × 5**、汎化は **40問 × 5**、確認は **50問 × 5**
- Shepard toneは各10秒
- 12音の無順序ペア66個 × 各4日 = 264日
- 追加順は E → F → A → Bb → D → Db → G → Ab → B → C → Eb → Gb
- 1ステージにつき新しいペアを1個だけ解禁し、直近6ペア / それ以前に分けて復習
- 4日間は「固定 → 比較 → 汎化 → 確認」で構成
- 解禁済みペア集合に含まれるペアだけが正解。それ以外はOOB
- OOB率は10〜15%。未解禁ペアのうち「既習音1つ＋未学習音1つ」だけを使い、トップノートは必ず既習音にする
- OOB枠の70%は現在ペア由来、30%は過去ペア由来（候補がない場合は利用可能な側へ再配分）
- 各ペアは ordered pair（どちらが低音か）×音域パターンをセルとして均等に出題
- 固定日は近接、比較日は近接＋1オクターブ超、汎化・確認日は近接＋1オクターブ超＋約2オクターブの広音程を使用
- 広音程を含む4オクターブ（2〜5）を固定のピアノ音色で出題

### 音源

- `smplr@0.20.0` の `SplendidGrandPiano` のみを使用
- velocity 90、duration 1.0秒、decayTime 0.8秒で固定
- リバーブ、EQ、コンプレッサーは不使用
- 1音版と2音版で使うC2〜B6（MIDI 36〜95）のサンプルをロード
- 初回再生時にライブラリとピアノサンプルをCDNから読み込むため、インターネット接続が必要

## プライバシーと保存

成績をサーバーへ送信せず、DB、Cookie、`localStorage`も使用しません。トレーニング中または終了画面から、その回の結果を任意でJSONファイルとして書き出せます（途中保存は `-partial` 付きファイル名）。音源ライブラリとピアノサンプルの取得時のみ外部CDNへ接続します。

## 注意

元研究は8週間で平均21.4時間・15,327試行でした。このアプリは研究の訓練原理を参考にしたWeb版であり、研究プロトコルの完全な複製や絶対音感の獲得を保証するものではありません。聴覚保護のため、小さめの音量から使用してください。
